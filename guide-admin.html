Flux manuel (demande d'accès par email)

  1. Vous recevez un email à jamal9@gmail.com avec nom, entreprise, email du demandeur.
  2. Ouvrez un terminal dans le dossier du projet.

💡 Depuis mai 2026, les utilisateurs peuvent créer eux-mêmes leur compte sur liya.digital/register. Un essai gratuit de 72h démarre automatiquement. Votre rôle est ensuite de confirmer les paiements cash et d'activer les abonnements payants.

🆕 Inscription autonome des utilisateurs

Les utilisateurs peuvent s'inscrire directement sur liya.digital/register sans intervention de votre part :

  • Formulaire : nom, entreprise (optionnel), email, mot de passe
  • Compte créé → connexion automatique → redirection vers le chat
  • Plan trial activé automatiquement : 72 heures, 50 000 tokens, Haiku & Sonnet uniquement
  • Après 72h, l'utilisateur voit une invitation à choisir un forfait payant depuis /billing

Votre rôle après inscription

  1. L'utilisateur choisit son plan depuis liya.digital/billing.
  2. Il clique 📩 Demander paiement cash → vous recevez un email automatique avec ses coordonnées, le plan choisi et le montant.
  3. Vous encaissez le paiement, puis allez dans l'admin : Administration → Entreprises → ⚙ Configurer → changez le plan et le quota.
  4. L'abonnement s'active immédiatement.
⚠️ Si l'utilisateur paie par Orange Money via l'interface (CinetPay), le plan est mis à jour automatiquement via le webhook. Aucune action de votre part n'est nécessaire dans ce cas.

💳 Gérer les paiements

Deux modes de paiement sont disponibles depuis la page liya.digital/billing :

1. Orange Money / Mobile Money (automatique)

  • L'utilisateur clique 🟠 Payer par Orange Money → redirigé vers CinetPay
  • Il paie par Orange Money, Wave, MTN, ou carte bancaire
  • Le webhook CinetPay met à jour automatiquement le plan — aucune action de votre part
  • L'utilisateur voit la confirmation sur la page billing avec le statut de paiement

2. Demande de paiement cash

  • L'utilisateur clique 📩 Demander paiement cash
  • Vous recevez immédiatement un email à jamal9@gmail.com avec : nom, entreprise, email, plan choisi, montant
  • Après encaissement, mettez à jour le plan manuellement (voir ci-dessous)

Activer un abonnement après paiement cash

  1. Connectez-vous sur liya.digital avec jamal@siaagilesolutions.com.
  2. Allez dans Administration → onglet Entreprises.
  3. Trouvez l'entreprise → cliquez ⚙ Configurer.
  4. Changez le Plan (ex : trial → premium) et ajustez le Quota de tokens.
  5. Cliquez Enregistrer — l'abonnement est actif immédiatement.
✅ Quand vous changez le plan via l'interface admin, la date planStartedAt est automatiquement mise à jour — ce qui démarre le compteur de 3 mois pour Découverte, ou efface le compte à rebours d'essai.

Enregistrer manuellement un paiement cash dans l'historique

Pour avoir une trace dans l'historique de facturation de l'entreprise (visible dans le dashboard) :

  1. Administration → Entreprises → ⚙ Configurer → section Paiements.
  2. Cliquez + Ajouter un paiement → saisissez le montant, la période et la méthode "cash".

⏳ Gérer les essais gratuits (plan trial)

Chaque compte créé via /register démarre automatiquement en trial avec :

  • 72 heures à partir de la création (trialEndsAt dans la base)
  • 50 000 tokens maximum
  • Haiku et Sonnet uniquement (Opus et Réflexion approfondie bloqués)

Ce que voit l'utilisateur

  • Bandeau jaune dans /billing : "⏳ X jours, Y heures restantes"
  • Après expiration : bandeau rouge "🚫 Essai expiré — Choisissez un forfait"
  • Si il essaie d'utiliser Opus : message "L'essai gratuit est limité à Haiku et Sonnet"

Prolonger un essai (si nécessaire)

npx tsx -e "
import 'dotenv/config';
import { prisma } from './lib/prisma';
const newExpiry = new Date(Date.now() + 3 * 24 * 60 * 60 * 1000); // +3 jours
prisma.tenant.update({
  where: { id: 'TENANT_ID_ICI' },
  data: { trialEndsAt: newExpiry }
}).then(t => console.log('Essai prolongé jusqu\'au', t.trialEndsAt))
  .finally(() => prisma.\$disconnect());
"

🔑 Réinitialiser un mot de passe

Méthode 1 — Interface web (recommandé)

  1. Connectez-vous sur liya.digital avec votre compte superadmin.
  2. Administration → Utilisateurs → trouvez l'utilisateur → 🔑 Modifier.
  3. Saisissez le nouveau mot de passe → Enregistrer.

Méthode 2 — Script (pour votre propre compte ou en cas de blocage)

EMAIL="utilisateur@entreprise.com" \
PASSWORD="NouveauMotDePasse123!" \
npx tsx prisma/reset-password.ts

Exemple de résultat :

✅  Mot de passe réinitialisé !

    Email      : utilisateur@entreprise.com
    Rôle       : admin
    Actif      : true
    Nouveau MDP: NouveauMotDePasse123!
🔴 Si votre propre compte superadmin est bloqué (comme aujourd'hui), c'est la seule méthode disponible. Le script met aussi active: true en cas de suspension accidentelle.

Réinitialiser le compte superadmin

EMAIL="jamal@siaagilesolutions.com" \
PASSWORD="VotreNouveauMotDePasse!" \
npx tsx prisma/reset-password.ts

🚫 Suspendre ou réactiver un utilisateur

Depuis l'interface web : Administration → Utilisateurs → bouton Actif/Suspendu.

Un utilisateur suspendu ne peut plus se connecter mais ses données sont conservées.

⚠️ Il est impossible de suspendre ou de modifier le mot de passe d'un compte superadmin depuis l'interface admin d'entreprise — seulement depuis un autre compte superadmin.

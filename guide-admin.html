📋 Guide Administrateur — Kamali

Procédure complète pour ajouter un utilisateur après réception d'une demande d'accès.

🔐 Usage interne — Ne pas partager

→ Flux de travail global

  1. Vous recevez un email de demande d'accès à jamal9@gmail.com avec le nom, l'entreprise et l'email du demandeur.
  2. Ouvrez un terminal et naviguez vers le dossier du projet.
  3. Listez les entreprises existantes pour trouver le bon TENANT_ID.
  4. Cas A — L'entreprise existe → ajoutez directement l'utilisateur.
    Cas B — Nouvelle entreprise → créez l'entreprise et le premier utilisateur.
  5. Communiquez l'email et le mot de passe temporaire à l'utilisateur par WhatsApp ou email sécurisé.
  6. Informez l'utilisateur de changer son mot de passe depuis le tableau de bord admin (Administration → Mot de passe).

⚙️ Prérequis — Ouvrir le terminal au bon endroit

Toutes les commandes doivent être exécutées depuis le dossier du projet. Copiez-collez exactement :

# Naviguer vers le projet
cd "/Users/papa/Documents/Business - EDS Solar/Claude Mali/assistai"

# S'assurer que Node 20 est utilisé
export PATH="/usr/local/opt/node@20/bin:$PATH"
💡 Conseil : ajoutez ces deux lignes à votre fichier ~/.zshrc ou ~/.bashrc pour ne pas avoir à les retaper à chaque session.

Lancez cette commande pour obtenir la liste complète avec les identifiants :

➕ Étape 2 — Ajouter l'utilisateur

Cas A — Entreprise existante
Cas B — Nouvelle entreprise

L'entreprise est déjà dans la liste. Il suffit d'ajouter l'utilisateur et de renseigner son TENANT_ID.

Option 1 — Variables en ligne de commande (recommandé)

Remplacez les valeurs en vert par les informations du demandeur :

EMAIL="ousmane@entreprise.com" \
NAME="Ousmane Diallo" \
TENANT_ID="COLLER_L_ID_ICI" \
PASSWORD="Bamako2026!" \
ROLE="user" \
npx tsx prisma/add-user.ts

Option 2 — Éditer le fichier directement

Ouvrez le fichier prisma/add-user.ts dans un éditeur, modifiez les constantes en haut du fichier, puis lancez :

npx tsx prisma/add-user.ts

Deux méthodes :

Méthode 1 — Interface web (recommandé)

  1. Connectez-vous sur assistai-six.vercel.app avec jamal@siaagilesolutions.com.
  2. Cliquez sur Administration dans la barre latérale.
  3. Onglet Utilisateurs → trouvez l'utilisateur → cliquez 🔑 Modifier.
  4. Saisissez le nouveau mot de passe → Enregistrer.

🚫 Suspendre ou réactiver un utilisateur

Depuis l'interface web : Administration → Utilisateurs → bouton Actif/Suspendu.

Un utilisateur suspendu ne peut plus se connecter mais ses données sont conservées.

🔧 Erreurs fréquentes et solutions

Message d'erreurCauseSolution
❌ Entreprise introuvable Le TENANT_ID est incorrect ou mal copié Relancez npx tsx prisma/list-tenants.ts et recopiez l'ID exact
⚠️ L'email est déjà utilisé Un compte existe déjà avec cet email Vérifiez dans Administration → Utilisateurs. L'utilisateur a peut-être oublié son mot de passe.
P1000: Authentication failed Le fichier .env est absent ou le mot de passe DB est incorrect Vérifiez que vous êtes dans le bon dossier et que .env existe
Cannot find module Node 20 non activé ou dépendances manquantes Relancez export PATH="/usr/local/opt/node@20/bin:$PATH" puis npm install
pyenv: pip: command not found Message inoffensif de pyenv — ignorez-le Ne rien faire — la commande s'est bien exécutée malgré ce message

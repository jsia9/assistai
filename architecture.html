🏗️ LIYA — Architecture Technique

liya.digital · Next.js 16 · Vercel · Supabase PostgreSQL · Anthropic Claude · Mai 2026

📎 Upload fichier — où est-il stocké ?

  • Le fichier binaire (PDF, Word, Excel, PPTX…) est reçu par une Vercel Serverless Function
  • Il est traité entièrement en mémoire RAM de la fonction : extraction du texte uniquement
  • Le binaire n'est jamais stocké — ni sur Supabase, ni sur S3, ni sur Supabase Storage
  • Le texte extrait est envoyé directement à Claude dans le message
  • Exception — documents de projet : le texte extrait est stocké dans la table ProjectDocument.content en Supabase (texte pur, pas le binaire)
💡 Si l'utilisateur uploade la même facture deux fois, elle est ré-analysée à chaque fois. Pas de déduplication.

⚖️ Limite de taille de fichier

  • Limite applicative : 4 Mo définie dans app/api/upload/route.ts
  • Limite Vercel : 4,5 Mo max par payload de fonction serverless — erreur 413 si dépassé
  • Le 4 Mo applicatif laisse 500 Ko de marge pour les headers HTTP + métadonnées
  • Si dépassé → message d'erreur en français : "Fichier trop volumineux (max 4 Mo)"
⚠️ Pour lever cette limite à 50+ Mo, il faudrait passer à des uploads directs browser → Supabase Storage (signed URLs). Le code contient déjà un commentaire préparatoire pour cette évolution.

📚 Contexte — comment et où est-il stocké ?

  • Stockage : table Message dans Supabase PostgreSQL (texte brut, colonne content TEXT)
  • À chaque message : les 20 derniers messages de la conversation sont rechargés depuis Supabase et envoyés à Claude
  • Pas de vector DB : pas d'embeddings, pas de RAG — recherche fulltext simple en DB
  • Modèle de données : Tenant → User → Conversation → Message (1-N-N-N)
  • La session utilisateur (NextAuth) est un JWT dans un cookie navigateur — pas en DB
💡 La limite de 20 messages = ~16 000 tokens de contexte. Au-delà, les plus anciens messages ne sont plus visibles par Claude dans la même conversation.

☁️ Y a-t-il Cloudflare ?

Non. Il n'y a pas de Cloudflare dans la stack actuelle.


  • CDN : Vercel Edge Network (PoPs dans 100+ villes)
  • DDoS / WAF : protection de base incluse dans Vercel Pro
  • SSL/TLS : certificat Let's Encrypt provisionné automatiquement par Vercel
  • Rate limiting : implémenté en code (3 req/IP/heure pour /register, 2/user/heure pour cash-request)
💡 Pour ajouter Cloudflare : pointer les DNS vers Cloudflare en mode proxy, puis ajouter Vercel comme origin. Donne WAF, règles de rate-limit avancées et analytics réseau.

🌐 liya.digital → Vercel ?

  • Oui — liya.digital est un domaine personnalisé mappé au déploiement Vercel
  • DNS configuré avec un CNAME : liya.digital → cname.vercel-dns.com
  • SSL automatique via Let's Encrypt (renouvelé par Vercel)
  • L'URL Vercel native (assistai-six.vercel.app) reste accessible en parallèle
  • Chaque git push main → nouveau déploiement Vercel en ~2 minutes → automatiquement servi sur liya.digital
💡 Les Preview Deployments Vercel (branches feature) ont leurs propres URLs *.vercel.app mais ne sont pas mappées sur liya.digital.

🔒 Sécurité en place

  • CSP headers (Content-Security-Policy) sur toutes les routes
  • HSTS (Strict-Transport-Security) 2 ans
  • X-Frame-Options: DENY (anti-clickjacking)
  • Rate limiting applicatif (register, cash-request)
  • Validation IDOR sur toutes les conversations
  • bcrypt (cost 12) pour les mots de passe
  • AuditLog append-only (login, CRUD, paiements)
  • Séparation tenant strict (toutes les requêtes DB filtrées par tenantId)
  • Pas de Cloudflare WAF à ajouter si trafic augmente
  • Pas de 2FA évolution possible
